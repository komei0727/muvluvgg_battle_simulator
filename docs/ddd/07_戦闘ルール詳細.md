# 戦闘ルール詳細

## 目的

本書は、[06\_戦闘状態遷移.md](./06_戦闘状態遷移.md) の各遷移から呼び出す判定・計算ルールを、実装とテストが可能な粒度で定義する。

各ルールには `R-領域-連番` 形式のルールIDを付ける。テストケース、ドメインイベントおよび後続の実装設計から、このIDを参照できるものとする。

## ルールの優先順位

競合する記述がある場合は、次の順に優先する。

1. [02\_仕様確認事項.md](./02_仕様確認事項.md) の決定事項
2. 本書の詳細ルール
3. スキル・効果定義に委ねられた項目の個別指定
4. [戦闘システム.md](./戦闘システム.md) の基本仕様
5. デフォルトルール

スキル・効果定義による個別指定は、ターゲット方式や効果量など、その定義に委ねられた項目だけを上書きする。最低1ダメージ、PSの1解決スコープ1回制限などの全体不変条件は、明示的に上書きを許可した仕様がない限り変更しない。

## 共通の数値規則

### R-NUM-01 割合の内部表現

- 設計書の100%を内部値 `1.0` として扱う。
- 25%加算は `+0.25`、5%減算は `-0.05` とする。
- ダメージの最終結果を求めるまで、途中値を丸めない。
- ダメージ以外のステータス計算値も途中で丸めず保持する。

### R-NUM-02 整数化

- 最終ダメージは全補正適用後に小数部分を切り捨てる。
- HP、AP、PP、EXゲージへ適用する値は、対応するルールで整数化済みの値とする。
- AP、PP、EXゲージを負数にしない。

### R-NUM-03 確率判定

- 確率判定は `RandomSource` を通して行う。
- 会心率のステータス値自体には上限・下限を設けない。
- 会心判定時の実効会心率だけを0～100%へ補正する。
- 0%は必ず失敗、100%は必ず成功とする。

## 編成の検証

### R-FRM-01 編成人数

- 味方・敵それぞれ1～5体とする。
- 0体または6体以上の編成は戦闘開始前に拒否する。

### R-FRM-02 配置

- 各陣営の盤面は横3列、縦2列とする。
- 列は絶対左から左、中央、右とする。
- 行は相手陣営に近い側を前列、遠い側を後列とする。
- 同じ陣営内で同じ配置位置を重複させない。

### R-FRM-03 ユニット重複

- 同じ `UnitDefinitionId` を同一陣営へ複数指定できる。
- 各参加枠へ異なる `BattleUnitId` を割り当てる。
- スキル状態、HP、効果、イベントログは `BattleUnitId` 単位で管理する。

### R-FRM-04 メモリー指定

- 各編成に0～6件のメモリーIDを指定できる。
- 存在しないメモリーIDまたは7件以上の指定を拒否する。
- メモリーの発動条件を満たさなくても入力エラーにはせず、効果を適用しない。

### R-FRM-05 規定ターン数

- 1～99の整数とする。
- 範囲外の値を戦闘開始前に拒否する。

## 共通座標と配置

### R-POS-01 共通座標への変換

列座標：

| 列       |   x |
| -------- | --: |
| 絶対左列 |   0 |
| 中央列   |   1 |
| 右列     |   2 |

行座標：

| 陣営・行 |   y |
| -------- | --: |
| 敵後列   |   0 |
| 敵前列   |   1 |
| 味方前列 |   2 |
| 味方後列 |   3 |

### R-POS-02 前方

- 味方の前方はy座標が減る方向とする。
- 敵の前方はy座標が増える方向とする。
- 絶対左列は陣営の向きにかかわらずx座標が小さい側とする。

### R-POS-03 距離

使用者 `(x1, y1)` と対象 `(x2, y2)` のマンハッタン距離を次で求める。

```text
distance = |x1 - x2| + |y1 - y2|
```

## 編成ボーナス

### R-BON-01 通常属性の役判定

編成が5体の場合、クレバーを除く属性構成について次の最も高い役を一つ適用する。

| 優先順位 | 役             | 条件                 | 攻撃力 |   HP |
| -------: | -------------- | -------------------- | -----: | ---: |
|        1 | ファイブカード | 同属性5体            |   +25% | +25% |
|        2 | フォーカード   | 同属性4体            |   +15% | +20% |
|        3 | フルハウス     | 同属性3体と別属性2体 |   +15% | +15% |
|        4 | スリーカード   | 同属性3体            |   +10% | +10% |

条件を満たさない場合、通常属性の編成ボーナスは0とする。4体以下の編成では役判定を行わない。

### R-BON-02 コミカル

1. 各コミカルをクレバー以外の属性として割り当てた全候補を評価する。
2. `ファイブカード > フォーカード > フルハウス > スリーカード > 役なし` の順で最も高い候補を採用する。
3. 採用した役の補正だけを適用する。
4. コミカルをクレバーとして扱わない。

### R-BON-03 クレバー

クレバーの人数が条件を満たしたすべての段階を累積する。

| 人数条件 | 追加補正           |
| -------- | ------------------ |
| 1人以上  | 防御力+30%         |
| 2人以上  | 攻撃力+10%、HP+10% |
| 3人以上  | 会心率+15%         |
| 4人以上  | 攻撃力+15%、HP+15% |
| 5人以上  | 攻撃力+25%、HP+25% |

例としてクレバー3人の場合、防御力+30%、攻撃力+10%、HP+10%、会心率+15%をすべて適用する。

## 戦闘中ステータス

### R-STA-01 基本式

対象ステータスごとに次を計算する。

```text
戦闘中ステータス =
  基本値
  × (1 + 編成補正 - 適性補正)
  × (1 + 戦闘中割合補正)
  + メモリー固定値補正
```

- 適正外配置によるHP、攻撃力、防御力の適性補正は `0.05` とする。
- 適正外配置の影響を受けないステータスでは適性補正を0とする。
- 編成補正、戦闘中割合補正、メモリー固定値補正は対象ステータスごとに集計する。

### R-STA-02 重複あり効果

- 同種の重複ありバフ・デバフをすべて有効とする。
- 有効な効果量を符号付きで加算する。
- メモリーの割合補正を重複ありバフとして加算する。

### R-STA-03 重複なし効果

- `EffectKindKey` が同じ重複なし効果を一つのグループにする。
- グループ内で効果の強度が最も大きい1件だけを計算へ採用する。バフは正の効果量、デバフは弱化量の大きさで比較する。
- 同じ絶対値の候補が複数ある場合も計算結果は同じため、効果インスタンスは変更せず最初に付与されたものを代表として扱う。
- 採用されない効果も、期間経過や解除に備えて保持する。

### R-STA-04 再計算

次の状態変更後、影響を受ける戦闘中ステータスを再計算する。

- バフ・デバフの付与、更新、解除、失効
- メモリー効果の有効・無効条件の変化
- その他、ステータス補正条件の変化

行動速度が変化した場合は、現在の1行動とPS連鎖完了後に未行動者を並べ直す。

## 行動順

### R-ORD-01 行動可能判定

戦闘可能であり、次のいずれかを満たすユニットを新しい行動順キューの候補とする。

- APが1以上
- EXゲージが満タン
- 凍結などで阻害されていないチャージ効果が発動待ち

### R-ORD-02 並べ替え

次の比較キーを上から順に適用する。

1. 現在の行動速度：降順
2. 陣営：味方、敵
3. 行：前列、後列
4. 列：絶対左、中央、右

この比較によって行動順を一意にする。

### R-ORD-03 行動種別予約

- キュー生成時点でEXゲージが満タンならEXを予約する。
- 満タンでなければASを予約する。
- キュー生成後にEXゲージが満タンになっても予約を変更しない。
- 速度変化による並べ替えでも予約を変更しない。
- チャージ効果発動は実効行動決定時にAS／EX予約より優先する。

### R-ORD-04 速度変化

1. 現在の1行動とPS連鎖を完了する。
2. キューから処理済み・戦闘不能のユニットを除く。
3. 残った未行動者を `R-ORD-02` で並べ直す。
4. 各予約のAS／EX種別を維持する。

## AS・EX・待機の選択

### R-ACT-01 実効行動の優先順

1. 気絶中：待機。チャージ中ならチャージをキャンセルする。
2. 凍結中：待機。チャージを維持する。
3. 発動可能なチャージ効果あり：チャージ効果を発動する。
4. EX予約：EXスキルを使用する。
5. AS予約：`R-ACT-02` でASを選ぶ。

### R-ACT-02 AS選択

ASを定義順に評価し、最初に次の条件をすべて満たしたものを選ぶ。

- クールタイムが0
- 必要APを保有
- 発動条件を満たす
- 必要な対象候補が1体以上存在する
- 気絶、凍結などによって使用を禁止されていない

候補がなければ待機する。

### R-ACT-03 リソース消費

| 行動                     | 消費           | EXゲージ増加 |
| ------------------------ | -------------- | ------------ |
| AS                       | スキル定義のAP | 消費APと同量 |
| PS                       | スキル定義のPP | 消費PPと同量 |
| EX                       | EXゲージ全量   | なし         |
| 通常待機                 | AP 1           | 1            |
| AP 0・EX満タンで行動不能 | EXゲージ全量   | なし         |
| チャージ効果発動         | なし           | なし         |

EXゲージ増加後はユニット固有の最大値で打ち止めとし、超過分を保持しない。

## ターゲット候補

### R-TGT-01 候補生成

1. スキル定義の対象陣営、対象種別、発動条件で候補を絞る。
2. 戦闘不能者を対象にできる明示的な指定がない限り、戦闘不能者を除く。
3. 自分自身だけを対象とするスキルでは使用者だけを候補にする。
4. 候補が0体なら、そのASを発動不能とする。

### R-TGT-02 デフォルト順

次の比較キーを上から順に適用する。

1. 使用者からのマンハッタン距離：昇順
2. 対象側の行：前列、後列
3. 対象の列：絶対左、中央、右

### R-TGT-03 最も遠い

`R-TGT-02` の並び全体を逆順にする。

### R-TGT-04 隣接

- 基準対象と同じ陣営の盤面内で、上下左右1マスを候補とする。
- 陣営境界を越えない。
- 斜め位置を含めない。

### R-TGT-05 目の前

- 基準対象と同じ陣営の盤面内で、同じ列の1マス前を候補とする。
- 基準対象が前列の場合は候補なしとし、スキルを発動不能とする。
- 陣営境界を越えない。

### R-TGT-06 列優先

1. 指定列にいる候補を先に並べる。
2. 左右列指定では、指定列からの列距離が小さい順に別列を並べる。
3. 前後列指定では指定列、反対列の順に並べる。
4. 同じ列優先度の候補は `R-TGT-02` で並べる。
5. 必要数に達するまで上から選ぶ。

右列優先の列順は右、中央、左となる。左列優先は左、中央、右となる。

### R-TGT-07 対象数不足

- 選択可能な候補が必要数より少ない場合、存在する候補だけを対象にする。
- 候補が1体以上ならスキル自体は発動する。
- 候補が0体ならAS選択時に発動不能として次のASを評価する。

### R-TGT-08 ステルス

1. 通常の対象順を決定する。
2. 第一優先対象がステルスを持つ場合に限り、ステルスを消費する。
3. その対象を候補順の最後へ移動する。
4. 第一優先対象でないステルス所持者は順序を変更しない。
5. 自分自身を対象とする自分のスキルには適用しない。
6. 条件によって対象が1体に限定されている場合は適用しない。

ステルス適用後に代替対象がいない場合はQ-TGT-05として保留中のため、そのケースを必要とするスキルは戦闘開始前に未対応として拒否する。

## スキル解決

### R-SKL-01 効果順

- `SkillDefinition` が持つ効果を定義順に解決する。
- 各効果の前後に必要なドメインイベントを発行する。
- イベントに反応したPS連鎖を直ちに解決してから、次の効果へ進む。
- 使用者が戦闘不能になった場合、未解決効果を中断する。
- 解決済みの効果を巻き戻さない。

### R-SKL-02 複数対象

- 選択済み対象をターゲット順に処理する。
- 対象ごとの効果適用後に発生したPSを直ちに解決する。
- 使用者が途中で戦闘不能になった場合、残りの対象へ効果を適用しない。

### R-SKL-03 複数ヒット

- 各ヒットを独立したダメージイベントとして処理する。
- 暗闇によるスキル単位のMISS結果は全ヒットで共有する。
- MISSでなければ、対象固有の特別な回避、会心、ダメージ、シールド、戦闘不能、PS連鎖をヒットごとに解決する。
- 使用者が途中で戦闘不能になった場合、残りのヒットを中断する。

### R-SKL-04 クールタイム設定

- スキル使用開始時にクールタイムを設定する。
- 設定した行動・ターンの終了時には減らさない。
- 次回以降の自身の行動終了またはターン終了で1ずつ減らす。
- 残数が0になった後の次の行動機会から使用できる。

### R-SKL-05 チャージ

- チャージ開始時に元スキルのコストを消費し、クールタイムを設定する。
- チャージ開始を一つの行動として完了する。
- 次の発動可能な行動機会に、AS／EX予約より優先してチャージ効果を発動する。
- チャージ効果発動ではAP・EXゲージを消費しない。
- チャージ効果発動を別の一つの行動とする。
- 気絶付与時にチャージをキャンセルする。
- 凍結中はチャージを維持し、解除後の次の行動機会に発動する。

## PS発動

### R-PS-01 発動タイミング照合

1. Battle Engineがドメインイベントを発行する。
2. `PassiveTriggerDefinition` のイベント種別とフェーズを照合する。
3. 発生源、対象、陣営、スキル種別などの条件を評価する。
4. 条件を満たしたPSを同じイベントの候補グループにする。

PSの発動タイミングは固定一覧に限定せず、新しいドメインイベントへ対応づけられるものとする。

### R-PS-02 候補順

候補グループ内を次の順で並べる。

1. 所有者の現在行動速度：降順
2. 味方陣営、敵陣営
3. 前列、後列
4. 絶対左、中央、右
5. 同じユニットではPS定義順

### R-PS-03 同時発動制限

- 同じイベントで候補になった、同時発動制限を持つPSだけを一つの制限グループにする。
- 先制攻撃候補を通常候補より優先し、その中で `R-PS-02` が最上位となるPS一つだけを候補に残す。
- 除外されたPSは発動済みとして記録しない。
- 別の発動タイミングで再度条件を満たした場合は、1解決スコープ1回制限に抵触しない限り候補になれる。

### R-PS-04 発動直前確認

候補を処理する直前に次を再確認する。

- 所有者が戦闘可能
- 所有者がチャージ中でない
- 必要PPを保有
- クールタイムが0
- 発動条件が現在も成立
- 現在の解決スコープで未発動

いずれかを満たさなくなった候補を破棄する。

### R-PS-05 発動と再入防止

1. `BattleUnitId + SkillDefinitionId` を発動済み集合へ追加する。
2. PPを消費し、消費量と同量だけEXゲージを増やす。
3. クールタイムを設定する。
4. `PassiveActivated` を発行する。
5. 効果を定義順に解決する。
6. 完了したら `PassiveResolved` を発行する。

発動済み集合への追加をPP消費やイベント発行より先に行い、再入を防ぐ。

### R-PS-06 新規候補の即時処理

- PS解決中に新しい候補グループが生じた場合、既存の未処理候補より先に処理する。
- 新しいグループを候補スタックの先頭へ積む。
- 新しいグループ内でも `R-PS-02` を適用する。
- 新しいグループの解決後、元のグループの続きへ戻る。

### R-PS-07 1解決スコープ1回

- ユニットの1行動では、同じPSを1回だけ発動できる。
- ターン開始・終了など行動外では、そのトップレベルイベントの解決スコープで1回だけ発動できる。
- 解決スコープ終了時に発動済み集合を破棄する。

### R-PS-08 先制攻撃

- 同じイベントの候補グループでは、先制攻撃特性を持つPSを通常のPS候補より先に処理する。
- 先制攻撃候補が複数ある場合は `R-PS-02` の行動速度、陣営、配置、定義順を適用する。
- 先制攻撃は発動条件、発動直前確認、同時発動制限、1解決スコープ1回制限を無視しない。
- 先制攻撃PSから新しい候補が生じた場合も `R-PS-06` に従い、新しい候補グループを直ちに処理する。
- 先制攻撃候補をすべて処理した後、同じ候補グループに残る通常候補へ戻る。

## 命中判定

### R-HIT-01 通常命中

- 通常の命中率・回避率を設けない。
- 暗闇または特別な回避効果がなければ必ず命中する。

### R-HIT-02 特別な回避効果

- 対象が有効な回避効果を持つ場合、その効果定義に従って回避を判定する。
- 攻撃が必中なら回避効果自体を発動させない。
- 対象がチャージ中なら自身の回避効果を発動させない。

### R-HIT-03 暗闇

1. スキル使用ごとに1回、使用者に付与された暗闇を付与順に取得する。
2. 各暗闇の指定確率でMISS判定を行う。
3. いずれか一つの暗闇でMISSになった場合、そのスキル全体をMISSとして扱う。
4. 複数対象・複数ヒットでも同じ判定結果を使用する。
5. 暗闇同士の確率を加算・乗算して一つの確率にまとめない。
6. 必中を持つスキルにも適用する。

MISSの場合、対象へのダメージと効果を適用しない。ただしMISSを契機とするPSは、対応するイベント定義があれば発動候補になれる。

## 会心判定

### R-CRT-01 実効会心率

```text
実効会心率 = min(100%, max(0%, 戦闘中会心率))
```

### R-CRT-02 会心倍率

会心が発生した場合、次の倍率を使用する。

```text
会心倍率 = 150% + 会心ダメージボーナス
```

会心が発生しない場合の倍率は100%とする。

## 属性相性

### R-ATR-01 有利属性

| 攻撃側       | 有利な防御側 |
| ------------ | ------------ |
| アグレッシブ | シャイ       |
| シャイ       | キュート     |
| キュート     | スマート     |
| スマート     | アグレッシブ |
| コミカル     | クレバー     |
| クレバー     | コミカル     |

### R-ATR-02 属性倍率

- 有利属性の場合は `125% + 属性相性ボーナス` とする。
- 有利属性でない場合は100%とする。
- 属性相性ボーナスは有利属性の場合だけ加算する。

## ダメージ計算

### R-DMG-01 基本ダメージ

```text
基礎ダメージ = max(0, 攻撃側の戦闘中攻撃力 - 防御側の戦闘中防御力)

計算ダメージ =
  基礎ダメージ
  × スキル威力
  × 属性倍率
  × 会心倍率
  × 被ダメージ倍率
```

### R-DMG-02 最終化

1. 全倍率適用後に小数部分を切り捨てる。
2. ダメージ無効効果がある場合も結果を1とする。
3. 計算結果が1未満の場合も1とする。
4. 最終ダメージを0未満にしない。

最低1ダメージは、ダメージを発生させる攻撃にだけ適用する。ダメージ効果を持たないスキルから1ダメージを発生させない。

### R-DMG-03 防御貫通

- 防御貫通はシールドとダメージ軽減効果を無視する。
- 基本ダメージ式の防御力は通常どおり参照する。
- 被ダメージ増加効果と属性・会心倍率は通常どおり適用する。
- 最終ダメージを直接HPへ適用する。

## シールドとHPへの適用

### R-SHD-01 シールドプール

- 物理シールド、ENシールド、タイプなしシールドを別々に保持する。
- 同じタイプのシールド付与値を加算する。
- 個別消滅条件を持つ付与元は `AppliedEffect` として保持し、有効な合計値を算出する。

### R-SHD-02 適用順

防御貫通でないダメージを次の順に適用する。

1. ダメージタイプに対応するタイプありシールド
2. タイプなしシールド
3. サブユニット
4. HP

対応しないタイプありシールドへダメージを適用しない。

### R-SHD-03 超過ダメージ

- シールド値を超えた残りを次の適用先へ渡す。
- HPを0未満にせず、超過分を破棄する。
- HPが0になった時点で戦闘不能処理を行う。

## サブユニット

### R-SUB-01 シールド効果

- 通常シールドをすべて適用した後にサブユニットがダメージを受ける。
- 毒、炎上など、通常シールドで受けられないダメージはサブユニットでも受けない。
- サブユニットの残HPをシールド表示値へ合算できるが、内部状態は通常シールドと分ける。
- シールド解除・シールド無視の対象とする。

### R-SUB-02 追加効果

- 所持者の攻撃対象ごとに追加ダメージを1ヒット加える。
- 複数対象への攻撃では、各対象へ1ヒットずつ加える。
- 追加デバフが定義されている場合も対象ごとに適用する。
- 追加ダメージでは通常の防御力減衰を行わない。

所持者の攻撃力が対象の防御力を下回る場合の特殊減衰式はQ-EFF-04として保留中である。この条件になり得るサブユニットを含む戦闘は、開始前に未対応として拒否する。

## リンクダメージ

### R-LNK-01 リンク元

- 対象へ算出された最終ダメージをリンク元の量とする。
- シールド・HPへの振り分け前の量を使用する。
- 元ダメージの物理／ENタイプと、シールド適用可否を引き継ぐ。

### R-LNK-02 適用

- リンクされた全対象へ、リンク元と同量のダメージをそれぞれ発生させる。
- 対象数で分割しない。
- リンク先では属性、会心、ダメージ増減効果を再計算しない。
- リンク先ごとに対応するシールドとHPへ適用する。
- 元ダメージが毒・炎上などシールド対象外なら、リンク先でもシールド対象外とする。

### R-LNK-03 再リンク禁止

- リンクダメージに `isLinkedDamage=true` を付ける。
- `isLinkedDamage=true` のダメージから新たなリンクを発生させない。

## 継続ダメージ

### R-DOT-01 共通

- 付与時に付与者の攻撃力をスナップショットとして記録する。
- 付与後の攻撃力変化や付与者の戦闘不能は計算へ影響しない。
- 付与対象の行動開始時に発生する。
- ダメージ軽減・増加、属性相性の影響を受けない。
- 効果定義の物理／ENダメージタイプを使用する。
- 各継続ダメージの最終結果で小数部分を切り捨て、ダメージが1未満なら最低1とする。

### R-DOT-02 固定継続ダメージ

- スナップショット攻撃力と効果定義の倍率から固定ダメージを算出する。
- 対応するタイプありシールド、タイプなしシールド、HPの順で適用する。

### R-DOT-03 炎上

- 最大3つまで保持する。
- 各炎上は付与時攻撃力、効果量、期間を保持する。
- 3つ重複した場合の2倍処理はQ-EFF-06として保留中である。

3重複へ到達し得る炎上を含む戦闘は、仕様確定まで未対応として拒否する。

### R-DOT-04 毒

```text
割合ダメージ = 現在HP × 毒効果率
上限ダメージ = 付与時攻撃力 × 100%
毒ダメージ = min(割合ダメージ, 上限ダメージ)
```

- 毒ダメージはシールドとサブユニットで受けない。
- 既存の毒へ再付与した場合、効果期間は長い方、効果量は大きい方を引き継いだ一つの毒を残す。
- 期間と効果量は別々の付与元から採用できる。

## 状態異常

### R-STS-01 共通

- 状態異常はデバフの一種とする。
- デバフ解除・デバフ無効の対象とする。
- 状態異常解除・状態異常無効は、状態異常として定義された効果だけを対象とする。

### R-STS-02 気絶

- AS、PS、EXスキルを新たに使用できない。
- 指定回数のアクティブ行動機会を待機すると解除する。
- APがあれば待機でAPを1消費する。
- AP 0・EX満タンならEXゲージを全量消費して待機する。
- 再付与時は残り回数が長い方を一つだけ残す。
- 付与時にチャージをキャンセルする。

### R-STS-03 凍結

- AS、PS、EXスキルを使用できない。
- 行動機会では待機する。
- 指定期間の経過、または新たな攻撃スキルによるダメージで解除する。
- 継続ダメージやデバフだけのスキルでは解除しない。
- 再付与時に期間延長や増幅率加算を行わない。
- チャージをキャンセルしない。
- 解除後の次の行動機会にチャージ効果を発動する。

凍結解除ダメージの増幅率はQ-EFF-05として保留中である。ダメージ解除を使用する凍結を含む戦闘は、仕様確定まで未対応として拒否する。

### R-STS-04 暗闇

- AS、PS、EXスキルに適用する。
- 必中を無視してMISS判定を行う。
- 複数の暗闇を付与順に独立して処理する。
- 暗闇同士の確率を合算しない。

## 効果期間と解除

### R-EFF-01 期間単位

効果定義は次の期間単位と初期回数を持つ。

- 行動単位：付与対象自身の行動終了時に減らす。
- ターン単位：ターン終了時に減らす。

効果インスタンスごとに、残り回数、期間単位、付与された行動ID、付与されたターン番号を保持する。特定イベントによる解除・消滅条件は、回数による効果期間とは別に保持する。

### R-EFF-02 解除

1. 解除スキルの対象カテゴリを確認する。
2. バフ、デバフ、状態異常、シールドなど一致する効果を抽出する。
3. 解除数や優先順が定義されている場合はその指定に従う。
4. `EffectRemoved` を発行する。
5. ステータス、シールド、行動速度を再計算する。

### R-EFF-03 無効

- 無効効果が有効な間は、対象カテゴリの新規付与を拒否する。
- すでに付与済みの効果を解除するかどうかは、無効効果の定義に明示する。
- 付与拒否もドメインイベントとして記録できるものとする。

### R-EFF-04 行動単位期間の減算

1. 対象自身の行動終了時に、その対象へ付与された行動単位効果を取得する。
2. 効果の付与行動IDが現在の行動IDと同じ場合は減らさない。
3. 次回以降の対象自身の行動終了時に、各効果インスタンスの残り回数を1減らす。
4. 対象以外のユニットの行動では減らさない。
5. 残り回数が0になった時点で即時に失効させ、`EffectExpired` を発行する。
6. 失効後、影響するステータス、シールド、行動速度を再計算する。

### R-EFF-05 重複効果の期間

- 重複あり効果は、同種であっても効果インスタンスと残り期間を個別に保持する。
- 重複なし効果も、既存効果を上書きせず個別に保持する。
- 重複なし効果では、その時点で最も強い1件だけを計算へ採用する。
- 採用中の最強効果が失効・解除された場合、残っている同種効果を再評価する。
- 残存効果があれば次に強い1件を即時に有効化する。
- 次点効果の残り期間は、非採用中も通常どおり個別に減算する。

### R-EFF-06 ターン単位期間の減算

1. ターン終了時にターン単位効果を取得する。
2. 効果の付与ターン番号が現在のターン番号と同じ場合は減らさない。
3. 次のターン終了時から、各効果インスタンスの残り回数を1減らす。
4. 残り回数が0になった時点で即時に失効させ、`EffectExpired` を発行する。
5. 失効後、影響するステータス、シールド、行動速度を再計算する。
6. 重複なしの最強効果が失効した場合は、残存する次点効果を即時に有効化する。

## 勝敗判定

### R-END-01 判定タイミング

- ユニットの1行動とPS連鎖完了後
- ターン開始・終了など、行動外のトップレベル解決スコープ完了後

### R-END-02 優先順

1. 敵味方とも全滅：味方勝利
2. 敵だけ全滅：味方勝利
3. 味方だけ全滅：味方敗北
4. 最終ターン終了かつ敵生存：味方敗北
5. 上記以外：継続

## ルールテストの基準

### 編成・ステータス

- コミカルの割り当て候補から最上位役を選べる。
- クレバー3人で1～3人条件の補正を累積する。
- 重複あり効果をすべて加算する。
- 重複なし効果では同種最大の1件だけを採用する。
- 適正外配置のHP、攻撃力、防御力だけを5%減らす。

### 行動・ターゲット

- 同速時に味方、敵、前列、絶対左列の順になる。
- 速度再計算でAS／EX予約を維持する。
- マンハッタン距離、前列、絶対左列で対象順が一意になる。
- 列優先で指定列が不足したとき隣の列から補う。
- 必要数より候補が少なくても存在する対象へ発動する。

### スキル・PS

- AP不足または対象不在のASを飛ばして次のASを選ぶ。
- 同じイベントでは先制攻撃PSを通常PSより先に処理し、先制攻撃同士を速度・陣営・配置・定義順で並べる。
- 新しく誘発されたPSを既存候補より先に解決する。
- 同じPSを1解決スコープで2回発動しない。
- 同時発動制限から最上位の1件だけを選ぶ。
- 使用者の戦闘不能後に残りのヒット・対象を処理しない。

### ダメージ・効果

- 攻撃力が防御力以下でも最低1ダメージになる。
- ダメージ無効でも1ダメージになる。
- 最終結果だけを切り捨てる。
- タイプあり、タイプなし、サブユニット、HPの順で適用する。
- リンク先全員へ同量を適用し、再リンクしない。
- 毒の割合ダメージを付与時攻撃力で上限制限する。
- 行動単位効果が付与行動では減らず、対象自身の次回行動終了から減る。
- 重複あり効果の期間を個別に減らす。
- 重複なしの最強効果失効後に、残存する次点効果が有効になる。
- ターン単位効果が付与ターンでは減らず、次のターン終了時から減る。
