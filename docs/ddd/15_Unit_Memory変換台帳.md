# Unit / Memory 変換台帳

## 目的

Issue #47（[Catalog] M2前提として残UnitとMemoryの基礎Catalogを整備する）の一環として、`raw/units/`・`raw/memories/` の全件を棚卸しし、production Catalog 候補（`catalog-src/`）への変換状況を機械的に追跡する。

前提文書: [`14_Catalog定義スキーマ.md`](./14_Catalog定義スキーマ.md)

## 件数サマリ

| 区分            | 総数 | 済み                               | 未変換 | 保留                        |
| --------------- | ---- | ---------------------------------- | ------ | --------------------------- |
| `raw/units/`    | 69   | 22（代表10 + 本Issue先行バッチ12） | 46     | 1（エレーナ・パステルコワ） |
| `raw/memories/` | 32   | 6（本Issue先行バッチ）             | 26     | 0                           |

代表10ユニットは Issue #41 / #46 で production Catalog（`catalog-src/units/`）へ昇格済み。本Issue #47 では「先行バッチ」として指定された12ユニット・6 Memory を同水準（`unit.json`/`skills.json`/`effects.json` フル変換）で追加した。残りは本Issueの棚卸し対象として一覧化するのみで、後続バッチ（追って起票するIssue）で変換する。

`unitDefinitionId` は `UNIT_<キャラクター名>_<衣装・バージョンを表す語>` の形式へ統一している（例: `UNIT_EVIE_ECO`、`UNIT_EVIE_KYONSHI`）。キャラクター名のみのID（例: 旧`UNIT_EVIE`）は使わない。

### 保留ユニットの理由

- **【心色見つめるムードメーカー】エレーナ・パステルコワ**: `raw/units/` 本文の HP・攻撃力・防御力・会心率・行動速度が `TBD` のままで、実データが未確定。基礎ステータスを推測値で埋めずに Catalog 化することはできないため、実データが判明するまで変換を保留する。

### 不完全変換の詳細（※印のユニット）

上表で「※不完全変換あり」と記載した6ユニットについて、raw記載の効果のうちCatalog v2の現行スキーマでは表現しきれず、近似または省略した箇所を一覧化する。いずれもM2の基礎Catalog項目（Unit ID・属性・タイプ・ロール・配置適性・基礎ステータス・AP/PP/EX関連値）には影響しない。スキル効果の完全表現は本Issueのスコープ外（Issue本文の「Out of scope」参照）であり、対応する`EffectActionDefinition`/`ConditionDefinition`/`TriggerDefinition`拡張が実装されるまで、後続Issueで解消する。

### 未変換 Memory の分類基準

M2 が必要とする静的補正・装備条件を優先し、後続バッチのスコープ判断に使えるよう次の観点で分類した。

| 分類                                | 該当件数 | 内容                                                                                                                                                                                                                                                                                                                          |
| ----------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 所属条件あり（affiliationId未定義） | 11       | `所属フィルタ`（例:「カオスメイデンに所属するキャラクター」）を持つが、Catalog側の `affiliationId` 採番方針が未確定。現行の production Catalog は全Unitの `metadata.affiliations` が空配列（[`14_Catalog定義スキーマ.md`](./14_Catalog定義スキーマ.md) の方針どおり）であり、affiliationId 体系を決めてから変換する必要がある |
| 静的補正のみ・変換容易              | 6        | attribute / unitType / role 単一フィルタ + 固定値または単純割合のstat補正のみで、`modifiers`省略記法または単純な `triggeredEffects` で機械的に変換できる                                                                                                                                                                      |
| 与ダメージ・被ダメージ補正あり      | 5        | `modifiers`省略記法では表現できない `APPLY_DAMAGE_MOD` が必要（対象がstatではなくdamage方向の補正のため）                                                                                                                                                                                                                     |
| TurnStarted発動を含む               | 2        | `BattleStarted` 以外の発動タイミング（ターン開始時）を持ち、`modifiers`省略記法の前提（BattleStartedのみ）から外れる                                                                                                                                                                                                          |
| 動的効果（Marker付与等）あり        | 1        | Marker付与を伴い、`EffectAction` 変換が必要                                                                                                                                                                                                                                                                                   |
| 敵側(ENEMY)対象の効果を含む         | 1        | 上記分類と重複するものを除き、Memoryが味方だけでなく敵側を対象とする効果を持つ                                                                                                                                                                                                                                                |

合計は重複分類を含むため26件と一致しないことがある（例: 所属条件 + 敵側対象の両方に該当するMemoryが1件ある）。

本Issue先行バッチで変換した6 Memory（`MEM_HARD_WARMUP`、`MEM_STRANGERS`、`MEM_TIMID_REINDEER_EVE`、`MEM_WALK_TOGETHER`、`MEM_HEART_COLOR`、`MEM_OJOSAMA_MANNERS`）はいずれも raw 記載の効果を近似・省略なくフル変換できている。

## 後続バッチへの申し送り

- 所属条件を持つ11件の Memory を変換するには、先に affiliationId の採番方針（`AFF_*` 等のprefixとIDリストをどのraw情報から起こすか）を決定する必要がある。
- `TurnStarted` 発動の Memory 2件は、`BattleStarted` 前提の `modifiers`省略記法が使えないため `triggeredEffects` のみで表現する。
- Marker付与を伴う Memory（お傍にいるのはいつでもピコですよ♪）は `APPLY_MARKER` の変換ルール（Marker ID採番、`14_Catalog定義スキーマ.md` の `MarkerDefinition` 制約）に従う。
- 残り46ユニットの変換は、代表10ユニット昇格時と同水準の精度（Issue #41/#44/#45/#46 相当）を要するため、5〜15ユニット程度のバッチに分割した後続Issueで進める。
- 「不完全変換の詳細」に挙げた表現ギャップ（自身/対象除外フィルタ、AOE内の対象別条件分岐、周期発動、クールタイムリセット、動的な被ダメージ軽減条件、位置関係trigger selector、同時発生イベント除外）は、Issue #44 のG-01〜G-10と同様に、対応するスキーマ拡張を検討する後続Issueの候補とする。
