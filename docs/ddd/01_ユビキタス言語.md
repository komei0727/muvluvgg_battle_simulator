# ユビキタス言語

## 目的

本書は、戦闘シミュレーションに関わる仕様、設計、実装、テストで使用する用語と意味を統一する。

定義が確定していない用語は、[02\_仕様確認事項.md](./02_仕様確認事項.md) の論点IDを併記する。

## システムと戦闘

| 用語                 | 定義                                                                                                                      |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 戦闘シミュレーション | APIリクエストで指定された味方編成、敵編成および規定ターン数を入力として、プレイヤー操作を介さず戦闘終了まで実行する処理。 |
| 戦闘                 | 戦闘開始から勝敗が確定するまでの状態と処理の総体。                                                                        |
| 味方陣営             | API利用者が指定する側の陣営。同時全滅時に勝者として扱う。                                                                 |
| 敵陣営               | 味方陣営と対戦する陣営。                                                                                                  |
| 編成                 | 最大5体のユニットと、それぞれを配置する盤面上の位置の組み合わせ。                                                         |
| 盤面                 | 各陣営がユニットを配置する横3列、縦2列の領域。両陣営の盤面を共通座標上で向かい合わせて扱う。                              |
| 配置位置             | 盤面上の列と前後列によって一意に定まり、共通座標へ変換できる位置。                                                        |
| 規定ターン数         | APIリクエストで指定される戦闘の上限ターン数。                                                                             |
| 勝利                 | 規定ターン以内に敵陣営が全滅した状態。同じ効果処理で両陣営が全滅した場合も味方陣営の勝利とする。                          |
| 敗北                 | 味方陣営が全滅した状態、または敵陣営を全滅させず規定ターンを終了した状態。                                                |
| 全滅                 | ある陣営に戦闘可能なユニットが1体も存在しない状態。                                                                       |
| 戦闘結果             | 勝敗、終了理由、終了ターン、最終状態、イベントログを含むシミュレーションの出力。                                          |
| イベントログ         | 戦闘中に発生した出来事を、発生順とその時点の状態が分かる粒度で記録したもの。                                              |

## 時間と進行

| 用語         | 定義                                                                                                                                                                                                                                                                                       |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ターン       | リソースの回復から始まり、戦闘可能な全ユニットに使用可能な行動がなくなるまで繰り返し行動する進行単位。                                                                                                                                                                                     |
| ターン開始   | APとPPの回復、ターン開始条件のパッシブスキル判定などを行うタイミング。                                                                                                                                                                                                                     |
| 行動順       | 敵味方を合わせた行動可能なユニットについて、行動速度の高い順に定めた順序。同速の場合は味方陣営、敵陣営の順とし、同陣営内では前列、絶対的な左列の順に優先する。                                                                                                                             |
| 行動順キュー | 現在の行動順と各ユニットが使用するスキル種別（ASまたはEX）を保持するキュー。ターン開始時とキューが空になった時点で生成し、先頭のユニットから1回ずつ行動してキューから除去する。                                                                                                            |
| 行動種別予約 | ターン開始時または現在のキューが空になった後、新しい行動順キューを生成するときに、その時点のEXゲージが満タンならEX、満タンでなければASとして、各ユニットの次の行動種別を固定すること。速度変化による行動順再計算や、予約後のEXゲージ変化では、現在のキューが保持する行動種別を変更しない。 |
| 行動機会     | 行動順に従って、ユニットがアクティブスキル、EXスキルまたは待機を実行できるタイミング。                                                                                                                                                                                                     |
| 行動開始     | 行動順キューから取り出されたユニットが、予約された行動を開始するタイミング。ASまたは待機ではAP、EXスキルではEXゲージを消費する。継続ダメージなどの発動基準にもなる。                                                                                                                       |
| 1行動        | 行動開始から、AS、EXスキル、待機、チャージ開始またはチャージ効果発動と、それらから連鎖したPSがすべて解決されるまで。チャージ開始とチャージ効果発動は、それぞれ別の1行動として数える。                                                                                                      |
| 待機         | スキルを使用できないユニットが行動機会を終了すること。通常はAPを1消費する。APが0かつEXゲージ満タンで行動順へ入り、状態異常などでEXを使用できない場合は、APの代わりにEXゲージを全量消費する。                                                                                               |
| 行動順再計算 | 行動速度が変化したとき、現在の行動順キューに残っている未行動ユニットだけを新しい速度順に並べ直すこと。各ユニットに予約済みの行動種別は再判定せず、そのまま保持する。                                                                                                                       |
| 周回         | 行動順キューの生成から、そのキュー内の全ユニットが1回ずつ行動してキューが空になるまでの進行単位。キューが空になっても使用可能な行動が残る場合は次の周回を開始する。                                                                                                                        |
| 戦闘終了判定 | 各1行動の終了後に、全滅または規定ターン数による勝敗を判定すること。                                                                                                                                                                                                                        |

## ユニットと能力

| 用語             | 定義                                                                                                                    |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------- |
| ユニット         | 戦闘へ参加するキャラクター。アプリケーション内のユニット定義を参照して生成する。                                        |
| ユニット定義     | 属性、タイプ、ロール、適正、基本ステータス、EXゲージ最大値およびスキルを保持するアプリケーション内のマスターデータ。    |
| 戦闘ユニット     | ユニット定義と配置を基に生成され、現在HP、リソース、効果など戦闘中に変化する状態を持つユニット。                        |
| 戦闘可能         | HPが1以上で、戦闘不能ではない状態。気絶や凍結は戦闘可能に含む。                                                         |
| 戦闘不能         | HPが0になり、新たな行動を行えない状態。行動順キューに残っている場合は即時に除去する。                                   |
| 基本ステータス   | HP、攻撃力、防御力、会心率、行動速度、属性相性ボーナス、会心ダメージボーナス、AP、PP。                                  |
| 戦闘中ステータス | `基本値 × (1 + 編成補正 - 適性補正) × (1 + 戦闘中割合補正) + メモリー固定値補正` で算出した値。各割合は小数表現とする。 |
| 属性             | アグレッシブ、スマート、シャイ、キュート、コミカル、クレバーのいずれか。                                                |
| ユニットタイプ   | ユニットが持つ物理、EN、敏捷のいずれかの分類。ダメージタイプとは別の概念である。                                        |
| ロール           | 物理アタッカー、ENアタッカー、タンク、サポート、コントロールのいずれか。                                                |
| 適正             | 前衛、後衛、またはその両方への配置適性。適正外配置ではHP、攻撃力、防御力が5%低下する。                                  |

## リソースとスキル

| 用語                   | 定義                                                                                                                                                                                                                                                                                                     |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AP                     | アクティブスキルごとに定められた量、または待機時に原則1を消費し、ターン開始時に全回復するリソース。                                                                                                                                                                                                      |
| PP                     | パッシブスキルごとに定められた量を発動時に消費し、ターン開始時に全回復するリソース。                                                                                                                                                                                                                     |
| EXゲージ               | APまたはPPの消費量に応じて蓄積するリソース。最大値はユニットごとに異なり、最大値を超えた増加分は保持しない。行動順キュー生成時に満タンならEX行動が予約され、EXスキル使用時に全量消費する。                                                                                                               |
| アクティブスキル（AS） | 行動機会に能動的に使用するスキル。ユニットに定義された上から順に、発動条件、クールタイム、AP残量などから使用可否を判定し、最初に使用可能なものを使う。                                                                                                                                                   |
| パッシブスキル（PS）   | 所定の発動条件を満たしたとき、PPを消費して発動するスキル。PSの発動から別のPSが誘発されることがあるが、同じ戦闘ユニットが持つ同じPSは1行動につき1回だけ発動できる。                                                                                                                                       |
| PS発動候補             | 同じタイミングで発動条件を満たしたPSの集合。所有者の行動速度が高い順に処理し、同速の場合は味方陣営、敵陣営の順、同陣営内では前列、絶対的な左列の順とする。同じユニットが持つ複数のPSについては、スキルの定義順に処理する。PSの解決中に新しい候補が生じた場合は、既存の未処理候補より先に直ちに処理する。 |
| PS発動済み集合         | 現在の1行動ですでに発動したPSを、所有する戦闘ユニットとPSの組み合わせで記録したもの。発動済みのPSを候補から除外し、PS連鎖の循環を防ぐ。                                                                                                                                                                  |
| EXスキル               | 行動順キュー生成時にEXゲージが満タンのユニットについて、ASの代わりに予約されるスキル。使用時にAPを消費しないため、APが残っていれば次回の行動順キューでASを使用できる。                                                                                                                                   |
| 発動条件               | スキルを使用または発動するために満たす必要がある条件。                                                                                                                                                                                                                                                   |
| 使用可能               | 発動条件、消費リソース、クールタイム、状態異常、対象の存在など、スキル使用に必要な条件を満たす状態。                                                                                                                                                                                                     |
| スキル解決             | 対象選択、命中判定、ダメージや効果の適用、誘発したPSの処理を規則に従って進めること。解決途中で使用者が戦闘不能になった場合は、未解決の効果を中断する。                                                                                                                                                   |
| クールタイム           | 使用済みスキルを再使用できるようになるまでに必要な行動数またはターン数。設定された行動またはターンの終了時には減らさず、次回以降の自身の行動終了時またはターン終了時に残数を減らす。                                                                                                                     |
| PS発動タイミング       | PSごとに定義する、発動条件を評価する契機。ターン開始、スキル使用前後、被ダメージ時などを表現でき、特定のタイミング一覧へ固定しない。Battle Engineで発生するドメインイベントと対応づける。                                                                                                                |
| チャージ               | スキル使用時に準備を開始し、原則として次の行動機会に効果を発動する状態。チャージ開始とチャージ効果発動はそれぞれ1行動として扱う。                                                                                                                                                                        |

## 対象選択

| 用語                 | 定義                                                                                                                                          |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | ------- | --- | ------- | ---------------- |
| ターゲット           | スキルの効果を適用するユニット。                                                                                                              |
| 共通座標             | 両陣営の盤面を同じ座標空間へ配置した座標。俯瞰時の絶対的な左から右へ `x=0,1,2`、敵後列、敵前列、味方前列、味方後列の順に `y=0,1,2,3` とする。 |
| 前方                 | 各陣営から相手陣営へ向かう方向。味方陣営ではy座標が減る方向、敵陣営ではy座標が増える方向とする。                                              |
| マンハッタン距離     | 共通座標上の使用者 `(x1, y1)` と対象 `(x2, y2)` について、`                                                                                   | x1 - x2 | +   | y1 - y2 | ` で求める距離。 |
| デフォルトターゲット | 使用者からのマンハッタン距離が小さい対象を優先し、同じ距離なら対象側の前列、次に盤面を俯瞰した絶対的な左列を優先する選択規則。                |
| 最も近い             | デフォルトターゲットの優先順で先頭になる対象。                                                                                                |
| 最も遠い             | デフォルトターゲットの優先順を逆にしたとき先頭になる対象。                                                                                    |
| 隣接                 | 同じ陣営の盤面内で、基準位置から上下左右1マスにある位置。陣営境界を越えた位置は含まない。                                                     |
| 目の前               | 同じ陣営の盤面内で、基準位置と同じ縦列にある1マス前の位置。基準位置が前列の場合、対象は存在せずスキルは発動しない。                           |
| 列優先               | 指定した前後列または左右列に属する対象を先に選び、不足時は近い別列から補う規則。                                                              |
| 対象数不足           | スキルが要求する対象数より選択可能な対象が少ない状態。スキルは不発にせず、存在する対象だけに発動する。                                        |
| 第一優先対象         | 対象選択規則によって最初に選択されたユニット。ステルスの発動判定に使用する。                                                                  |

## 効果とダメージ

| 用語               | 定義                                                                                                                                                                                                                        |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 効果               | スキルなどによりユニットまたは戦闘へ与えられる状態変化の総称。                                                                                                                                                              |
| 効果期間           | バフ、デバフ、状態異常などの効果が有効である残り回数。行動単位またはターン単位を持ち、効果インスタンスごとに管理する。                                                                                                      |
| 行動単位効果期間   | 効果を付与された対象自身の行動終了時に残り回数を1減らす期間。対象以外のユニットの行動では減らさず、付与された当該行動でも減らさない。                                                                                       |
| ターン単位効果期間 | ターン終了時に残り回数を1減らす期間。付与されたターンの終了時には減らさず、次のターン終了時から減らす。                                                                                                                     |
| 効果失効           | 効果期間の残り回数が0になった時点で、その効果インスタンスを無効にすること。失効後、ステータスや有効効果を即時に再計算する。                                                                                                 |
| バフ               | 自陣営を有利にする、または妨害を防ぐ効果。各バフは重複ありまたは重複なしの区分を持つ。                                                                                                                                      |
| デバフ             | 対象を不利にする効果。状態異常を含み、各デバフは重複ありまたは重複なしの区分を持つ。                                                                                                                                        |
| 重複あり効果       | 同種の効果が複数存在するとき、そのすべてを有効として効果量を加算するバフまたはデバフ。各効果インスタンスの期間は個別に管理する。                                                                                            |
| 重複なし効果       | 同種の効果が複数存在するとき、効果量が最も大きい1件だけを有効とするバフまたはデバフ。各効果インスタンスを個別に保持し、最強効果の失効後は次に強い効果を有効にする。                                                         |
| 戦闘中割合補正     | 有効な重複あり効果の全件と、同種ごとに選んだ重複なし効果の最大1件を加算した値。メモリーの割合補正も重複ありバフとしてこの値へ加える。                                                                                       |
| 状態異常           | 特殊な振る舞いを持つデバフ。気絶、炎上、毒、凍結、暗闇が定義されている。                                                                                                                                                    |
| ダメージタイプ     | 発生するダメージが持つ物理ダメージまたはENダメージの分類。ユニットタイプとは独立してスキルや効果に定義する。                                                                                                                |
| 基礎ダメージ       | 攻撃力から防御力を引いた値。結果が0未満の場合は0とする。                                                                                                                                                                    |
| 最終ダメージ       | 基礎ダメージへスキル威力、属性相性、会心、被ダメージ補正などを適用し、最後に小数部分を切り捨てた値。ダメージを発生させる攻撃では、計算値やダメージ無効効果にかかわらず最低1とする。                                         |
| 実効会心率         | 会心判定に使用する確率。ステータスとしての会心率には上限・下限を設けないが、判定時には0%未満を0%、100%超を100%として扱う。                                                                                                  |
| 通常命中           | 特別な回避効果や暗闇などがなければ、攻撃が必ず命中すること。通常の命中率・回避率は設けない。                                                                                                                                |
| シールド           | 対応するダメージをHPより先に肩代わりする値。物理、EN、タイプなしの3種類のプールを持ち、同じタイプの付与値は加算する。物理またはENダメージでは対応するタイプありシールドを先に消費し、その後にタイプなしシールドを消費する。 |
| サブユニット       | シールドとしての耐久値と、所持者の攻撃へダメージやデバフを追加する能力を併せ持つバフ。通常のシールドと同時に存在する場合は、通常のシールドを先に消費する。                                                                  |
| ステルス           | 自身が第一優先対象に選ばれたときに消費し、対象選択上の優先度を最後へ下げるバフ。                                                                                                                                            |
| 継続ダメージ       | 付与対象の行動開始時に発生する固定ダメージ効果。付与時に付与者の攻撃力を記録し、その値を以後のダメージ計算に使用する。                                                                                                      |
| リンク             | 対象に発生したダメージと同量のリンクダメージを、リンクされた全対象へそれぞれ発生させる効果。対象数で分割せず、リンクにより生じたダメージは再リンクしない。                                                                  |
| 最低ダメージ       | ダメージを発生させる攻撃について保証される1ダメージ。基礎ダメージが0の場合や、対象がダメージ無効効果を持つ場合にも発生する。                                                                                                |
| 防御貫通           | シールドおよびダメージ軽減効果を無視してダメージを与える特性。                                                                                                                                                              |
| 必中               | 特別な回避効果を発動させず命中する特性。ただし暗闇によるMISS判定は受ける。                                                                                                                                                  |
| 先制攻撃           | 通常の行動順より優先して行動する特性。複数存在するときは行動速度順とする。                                                                                                                                                  |
| 同時発動制限       | 同じタイミングで発動可能な対象スキルのうち、行動速度が最も高い所有者のスキル1つだけを発動させる制約。                                                                                                                       |

## 編成関連

| 用語                 | 定義                                                                                                                                                                                        |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 編成ボーナス         | 5体の属性構成に応じて、戦闘開始時の能力へ適用する補正。コミカルはクレバー以外の属性のうち最も高い編成ボーナスになる属性として自動評価する。クレバー人数ボーナスは到達した全段階を累積する。 |
| メモリー             | APIリクエストで各編成へ最大6件指定し、条件を満たすユニットや位置へ効果を与える定義データ。固定値補正と割合補正があり、割合補正は重複ありバフとして戦闘中割合補正へ加える。                  |
| ポジション適性不一致 | ユニットを適正のない前後列に配置した状態。                                                                                                                                                  |

## 使用しない表現

- APを「ASポイント」、PPを「PSポイント」と呼ばない。
- 「攻撃順」は、攻撃以外の行動も含むため「行動順」と呼ぶ。
- 「キャラクター」と「生徒」は、戦闘モデル上では「ユニット」に統一する。
- 1回のAS発動だけを「1行動」と呼ばない。誘発したPSの解決完了までを含める。
