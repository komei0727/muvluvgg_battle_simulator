# 不完全変換対応予定方針

## 目的

`15_Unit_Memory変換台帳.md` の不完全変換一覧に、各項目をいつ・どの機能領域で解消するかを追跡できる情報を追加する。

不完全変換は、発見されたIssueや変換バッチ単位ではなく、解消可能になる最初のエンジン縦切りに紐づける。

## 追加する列

不完全変換の詳細表に、次の2列を追加する。

```markdown
| unitDefinitionId | 該当スキル | 変換できなかった内容 | 原因 | 対応予定 | 対応テーマ |
| ---------------- | ---------- | -------------------- | ---- | -------- | ---------- |
```

## 対応予定

`対応予定` には、原則として該当ギャップを解消できる最初のマイルストーンを記載する。

| 対応予定  | 対象                                                                                                                                                       |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `M2-DATA` | 所属条件、配置・属性・基礎ステータスなど、M2の開始状態生成に直接関わるデータ不備                                                                           |
| `M5`      | AP/EX消費、クールタイム開始・短縮・リセット、チャージ開始/解除など、行動ライフサイクルそのもの                                                             |
| `M6`      | PS発動条件、イベント照合、PP消費、PS発動回数、N回ごと発動、同時発動制限、発動除外条件                                                                      |
| `M7`      | 効果インスタンス、期間、重複、解除、状態異常、効果無効、ステータス再計算、高度ターゲット、Memory triggeredEffects、バフ/デバフ判定、リソース獲得量Modifier |
| `M8`      | ダメージpipeline、会心後分岐、防御貫通、シールド、サブユニット、リンク、反射、継続ダメージ、毒、HP割合スケール                                             |
| `M9`      | production Catalog 全体の最終承認、未実装Capability拒否の総点検                                                                                            |

`M6`のPS発動条件には、発動回数・N回ごと・累計ダメージ閾値を判定するための最小`RuntimeCounter`（`SkillRuntime`スコープ）を含む。`Battle`／`BattleUnit`スコープは当初M6の対象としていたが、利用するproduction Catalog定義が現れるまでFeature Complete必須対象に含めず、Catalogロード時点で明示的に拒否する契約とした（レビュー再レビュー[P2]、Issue #143）。必要な定義を追加する際は、そのproduction経路と同じTaskで実装・検証する。`AppliedEffect`スコープの`RuntimeCounter`（効果インスタンス自身に所有される汎用Counter管理）は`M7`のEFF-005（Issue #162）で実装した — `DurationDefinition.counterUpdates`（`APPLIED_EFFECT`スコープ）と`expiration.conditions`（R-EFF-08）からの参照で構成する。`MarkerState`（consumption/expiration機構が別途未実装）と`EffectSequence`（対応するResolverが未実装）はいずれもCatalogロード時点で`counterUpdates`／`RuntimeCounter`スコープを明示的に拒否する。いずれも同じ`RuntimeCounter`モデルの異なるスコープであり、モデル自体の追加マイルストーンではない。詳細は [`05_ドメインモデル.md`](./05_ドメインモデル.md)「RuntimeCounter」と [`13_実装計画.md`](./13_実装計画.md)「M6 PSイベントエンジン」を参照。

Unitスキルの不完全変換は、基本的に `M2` には割り当てない。現状の不完全変換は、M2の基礎Catalog項目には影響しないためである。

## 対応テーマ

`対応テーマ` には、Issue化や横断修正に使える短い分類名を記載する。

同じ原因で複数スキルに発生している不完全変換は、同じ対応テーマに寄せる。

| 対応テーマ                              | 対応予定 | 内容                                                                                                                      |
| --------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------- |
| `TARGET_EXCLUDE_RESOLVED_UNIT`          | `M7`     | 自身を除く味方全体、対象を除く敵全体など、解決済み対象から特定Unitを除外する                                              |
| `TARGET_ORDER_MARKER_COUNT`             | `M7`     | Marker所持数が最多/最少の対象選定                                                                                         |
| `TARGET_ORDER_STAT_LOWEST_OR_SPEED`     | `M7`     | 最も攻撃力が低い対象、最も行動速度が速い対象、最大HPが最も高い対象など、統計値の極値でのTargetOrderKey欠落                |
| `AOE_PER_TARGET_CONDITION`              | `M7`     | AOE対象ごとに状態・Marker有無で効果分岐する（対象範囲が空の場合の不発条件を含む）                                         |
| `RUNTIME_COUNTER_MODULO`                | `M6`     | AS/PS/会心などがN回ごとに発動する                                                                                         |
| `TRIGGER_POSITION_RELATION`             | `M6`     | 自身の目の前の味方など、位置関係を条件にするPS発動                                                                        |
| `TRIGGER_PAYLOAD_FILTER`                | `M7`     | デバフ付与時、特定状態異常付与時など、イベントpayloadを細かく絞り込む                                                     |
| `TRIGGER_EXCLUSION_TIMING`              | `M6`     | 戦闘開始時・ターン開始時・ターン終了時には発動しない、などの除外条件                                                      |
| `COOLDOWN_MANIPULATION`                 | `M5`     | 他スキルのクールタイムをリセット・短縮する                                                                                |
| `RESOURCE_GAIN_MOD`                     | `M7`     | 一定期間、EXゲージなどの獲得量を割合増減させる                                                                            |
| `HP_DIRECT_COST`                        | `M7`     | 現在HP/最大HPを直接消費するコスト                                                                                         |
| `REMOVE_BUFF_CATEGORY`                  | `M7`     | `REMOVE_EFFECTS` でバフカテゴリを解除する                                                                                 |
| `LINKED_EFFECT_GROUP`                   | `M7`     | 元効果の解除・付与者死亡に連動して別効果を解除する                                                                        |
| `DAMAGE_LINK`                           | `M8`     | 被ダメージ共有、転送、リンク追跡                                                                                          |
| `REFLECT_DAMAGE`                        | `M8`     | 受けたダメージの反射                                                                                                      |
| `SUBUNIT_DURATION`                      | `M8`     | サブユニットの存続期間                                                                                                    |
| `POST_DAMAGE_CRITICAL_BRANCH`           | `M8`     | この攻撃で会心した場合の追加効果                                                                                          |
| `POST_DAMAGE_SURVIVAL_BRANCH`           | `M8`     | 攻撃後に対象が生存/撃破していた場合の追撃・追加効果                                                                       |
| `DYNAMIC_DAMAGE_MOD_CONDITION`          | `M8`     | 攻撃対象や攻撃者の現在HP割合・Marker所持などで与/被ダメージ補正を動的評価する                                             |
| `HP_RATIO_SCALE_FORMULA`                | `M8`     | HPが多い/少ないほど効果量が変わるダメージ・補正                                                                           |
| `CONTINUOUS_DAMAGE_OR_POISON_ADVANCED`  | `M8`     | 毒、炎上、継続ダメージの特殊統合・上限・snapshot                                                                          |
| `CONFUSION_OR_DAMAGE_TO_HEAL`           | `M8`     | 混乱、幻惑など、攻撃対象やダメージ処理を大きく変える状態                                                                  |
| `UNREACHABLE_BRANCH_BY_RAW_DATA`        | `M9`     | rawに分岐条件だけ存在するが、現行raw内に到達手段がないもの                                                                |
| `TARGET_STATE_QUERY_BUFF_DEBUFF`        | `M7`     | 対象が毒/炎上/バフ/デバフなど何らかの状態を持つかを汎用的に判定する`ConditionDefinition`/`TARGET_STATE.field`が存在しない |
| `TARGET_FILTER_MARKER_COUNT_THRESHOLD`  | `M7`     | `TargetFilterDefinition`がMarker所持「数」のしきい値条件を持てない（`HAS_MARKER`は所持の有無のみ）                        |
| `CUMULATIVE_DAMAGE_THRESHOLD_TRIGGER`   | `M6`     | 累計ダメージが閾値を超えるたびに周期発火する`ConditionDefinition`が存在しない                                             |
| `DYNAMIC_DURATION_ON_REAPPLY`           | `M7`     | 既存効果の残存有無に応じて、再付与時の`duration`を動的に変える仕組みが存在しない                                          |

| `SUBUNIT_ADDITIONAL_DAMAGE_DEBUFF`      | `M8`     | `APPLY_SUBUNIT.additionalDamage`がダメージのみを表現でき、付随するデバフを持てない                                        |
| `TARGET_ORDER_UNITTYPE_OR_SELF_EXCLUDE` | `M7`     | unitType優先や自身除外優先を表す`TargetOrderKey`が存在しない                                                              |
| `TARGET_FILTER_MARKER_BY_AREA`          | `M7`     | 列などの`AreaDefinition`単位でMarkerの所在を判定する`TargetFilter`が存在しない                                            |
| `STACK_LIMIT_ON_STAT_MOD`               | `M7`     | `APPLY_STAT_MOD.stacking`にMarker同様の`stack.max`相当の重複上限フィールドが存在しない                                    |
| `EFFECT_IMMUNITY_STATUS_GRANULARITY`    | `M7`     | `EFFECT_IMMUNITY`が状態異常の種類単位（気絶のみ等）を指定できず、`STATUS`カテゴリ全体でしか免疫を表現できない             |
| `ON_ATTACK_BONUS_DAMAGE_BUFF`           | `M7`     | 対象の攻撃を起点に追加ダメージを発生させる汎用バフを表す`EffectAction`が存在しない                                        |
| `SET_THRESHOLD_ACTIVATION_CONDITION`    | `M7`     | 対象集合の一部がしきい値を満たすかを判定し、満たさない場合にスキル自体を不発にする`ConditionDefinition`が存在しない       |
| `REMOVE_EFFECTS_CATEGORY_GAP`           | `M7`     | `REMOVE_EFFECTS`の`categories`に`SHIELD`・`SUBUNIT`など、`DEBUFF`/`BUFF`以外の効果カテゴリが存在しない                    |
| `REMOVE_EFFECTS_COUNT_LIMIT`            | `M7`     | `REMOVE_EFFECTS`/`REMOVE_MARKER`が解除件数の上限・指定数を指定できない                                                    |

`TARGET_STATE_QUERY_BUFF_DEBUFF` 以降の16テーマは、Issue #54 の最終レビューで `15_Unit_Memory変換台帳.md` 全113行に本方針を適用した際、既存23テーマ（`TARGET_EXCLUDE_RESOLVED_UNIT`〜`UNREACHABLE_BRANCH_BY_RAW_DATA`）のいずれでも表現できなかったため追加した。

## 運用ルール

- `対応予定` は、発見されたIssueではなく、解消可能になる最初のマイルストーンを書く。
- Unitスキルの不完全変換は、原則として `M2` には割り当てない。
- 同じ技術的原因のものは、同じ `対応テーマ` に寄せる。
- 1つの不完全変換に複数原因がある場合は、主要なブロッカーを `対応テーマ` に書く。
- 複数マイルストーンにまたがる場合は、Catalog近似を解除できる最終側のマイルストーンを優先する。
- Capability実装時は、同じ `対応テーマ` の行をまとめて確認する。
- 対応テーマを新設する場合は、既存テーマで表現できないことを確認してから追加する。
- 解消した行は削除せず、必要なら別途「解消済み」扱いに移すか、該当PRで台帳から除去する方針を決める。
- `M9` は最終検証・承認用であり、新規表現力の主担当には原則しない。
